認証フロー設計

ダミー認証 × JWT × Zustand 永続化 × middleware ルートガード

ログインフロー

1
/login でメール入力
admin@demo.com または user@demo.com を入力してログインボタン押下
2
POST /api/v1/auth/login をコール
FastAPI がメールアドレスでロール判定 → JWTトークンを発行して返す
3
Zustand store に保存
token / email / role を store に保存 → localStorage に永続化（リロード後も維持）
4
/dashboard にリダイレクト
ロールによって admin / user の表示を切り替え

ルートガード（middleware.ts）

🚫 未ログイン状態で /dashboard にアクセス
middleware が token の有無を確認
→ なければ /login にリダイレクト
✅ ログイン済みで /login にアクセス
middleware が token を確認
→ あれば /dashboard にリダイレクト
🔒 admin 専用ページへ user でアクセス
middleware または各ページで role を確認 → /dashboard にリダイレクト（権限なし）

Zustand Auth Store の構造

interface AuthState {
  token: string | null
  email: string | null
  role: 'admin' | 'user' | null
  login: (token, email, role) => void
  logout: () => void
  isAdmin: () => boolean
}

データフロー設計

TanStack Query（サーバー状態）× Zustand（クライアント状態）の役割分担

状態管理の役割分担

⚡ TanStack Query
担当：サーバーデータ
• 車両一覧 / 予約一覧
• ダッシュボードデータ
• ユーザー一覧

自動でやってくれること：
• キャッシュ管理
• ローディング / エラー状態
• 更新後の自動再フェッチ
• 楽観的更新
🐻 Zustand
担当：クライアント状態
• ログイン中ユーザー情報
• JWTトークン
• ユーザーロール（admin/user）

特徴：
• localStorage に永続化
• ページリロード後も維持
• どこからでもアクセス可

API呼び出しの流れ

🖥️ Page / Component
→
useQuery / useMutation を呼ぶ
⚡ TanStack Query
→
features/cars/hooks/useCars.ts などのカスタムフック経由
🌐 lib/api.ts
→
fetch ラッパー。Authorization ヘッダーに JWT を自動付与
🐻 Zustand store
→
api.ts が token を取得。認証情報は store から読む
🔌 FastAPI Backend
←
レスポンスが返る → TanStack Query がキャッシュして画面に反映

カスタムフックの例（features/cars/hooks/useCars.ts）

export function useCars() {
  return useQuery({
    queryKey: ['cars'],
    queryFn: () => apiClient.get('/cars'),
  })
}

export function useCreateCar() {
  const queryClient = useQueryClient()
  return useMutation({
    mutationFn: (data) => apiClient.post('/cars', data),
    onSuccess: () => queryClient.invalidateQueries(['cars']),
  })
}

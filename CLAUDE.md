# CarDelivery - 営業車配車管理システム

## プロジェクト概要

ポートフォリオ用のWebシステム。営業車の配車スケジュール・車両・予約を一元管理する。
クラウドワークス案件獲得を目的としたポートフォリオ作品。

## 技術スタック

| 役割 | 技術 |
|------|------|
| Frontend | Next.js 15 + React 19 + TypeScript |
| Backend | Python 3.12 + FastAPI |
| ORM | SQLAlchemy 2.0 + Alembic |
| DB | PostgreSQL (ローカル開発 / 本番 AWS RDS) |
| 認証 | ダミー認証（ポートフォリオ用・JWT形式で見た目は本物） |
| Deploy | AWS (Amplify + App Runner + RDS) |
| スタイル | Tailwind CSS + shadcn/ui |

## ユーザーロール

| ロール | 権限 |
|--------|------|
| admin | 全予約の承認・却下・車両管理・ユーザー管理 |
| user | 自分の予約申請・自分の予約履歴確認 |

## 主要機能

### 一般社員（user）
- 配車予約申請（車両・日時・行先・目的）
- 自分の予約一覧・ステータス確認
- ダッシュボード（今日の配車状況 / 自分の予約履歴）

### 管理者（admin）
- 予約承認・却下
- 未承認申請一覧（ダッシュボードに表示）
- 車両管理（登録・編集・削除）
- 走行距離の記録・管理

## 承認フロー

```
申請（pending）→ 管理者承認（approved）→ 使用中（in_progress）→ 完了（completed）
                 管理者却下（cancelled）
```

## ダッシュボード表示項目

- 今日の配車状況（誰がどの車でどこへ）
- 自分の予約履歴
- 未承認申請一覧（adminのみ）
- 車両稼働率（adminのみ）

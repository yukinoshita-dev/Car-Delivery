# CarDelivery - 機能仕様書

## 対象ユーザーとロール

| ロール | 説明 |
|--------|------|
| admin | 配車承認・車両管理・全予約管理を行う管理者 |
| user | 配車申請・自分の予約確認を行う一般社員 |

---

## 認証

ポートフォリオ用のダミー認証。クライアントがそのまま動かせることを優先。

- ログイン画面はメール + パスワード入力フォーム（見た目は本物）
- パスワード検証なし。以下のデモアカウントでメールが一致すればログイン成功
- JWTトークン形式でフロントに渡し、ロールベースの画面制御を行う

| メール | ロール |
|--------|--------|
| admin@demo.com | admin |
| user@demo.com | user |

## 機能詳細

### 1. ダッシュボード

#### user向け
- **今日の配車状況**: 本日の承認済み予約一覧（車両・予約者・行先・時間）
- **自分の予約履歴**: 直近の予約5件（ステータス付き）

#### admin向け（userの内容に追加）
- **未承認申請一覧**: pending状態の予約をカード形式で表示。ワンクリックで承認・却下
- **車両稼働率**: 各車両の今月の予約件数・走行距離の集計グラフ

---

### 2. 配車予約申請

**入力項目**

| 項目 | 必須 | 備考 |
|------|------|------|
| 車両 | ○ | 選択時に空き状況を表示 |
| 利用開始日時 | ○ | 日付・時間ピッカー |
| 利用終了日時 | ○ | 開始より後の時間のみ選択可 |
| 行先 | ○ | テキスト入力 |
| 利用目的 | ○ | テキスト入力 |

**バリデーション**
- 同じ車両・同じ時間帯に既存の承認済み予約がある場合はエラー（重複チェック）
- 終了日時は開始日時より後であること

**申請後のステータス遷移**

```
申請 → pending → admin承認 → approved → 利用開始 → in_progress → 返却・距離入力 → completed
                → admin却下 → cancelled
```

---

### 3. 予約ステータス

| ステータス | 表示名 | 説明 |
|------------|--------|------|
| pending | 承認待ち | 申請済み・未対応 |
| approved | 承認済み | 管理者が承認 |
| in_progress | 使用中 | 利用開始日時を過ぎた承認済み予約 |
| completed | 完了 | 返却・走行距離入力済み |
| cancelled | 却下 | 管理者が却下 |

---

### 4. 車両管理（admin）

**車両情報**

| 項目 | 説明 |
|------|------|
| 車両名 | 例：プリウス1号 |
| ナンバープレート | 例：品川 300 あ 1234 |
| 定員 | 乗車可能人数 |
| 現在の使用状況 | 空き / 使用中 / メンテナンス中 |
| 累計走行距離 | 予約完了時に加算 |

**操作**
- 車両の新規登録・編集・削除
- ステータス変更（メンテナンス中など）

---

### 5. カレンダービュー

- 月・週単位のカレンダーで予約を表示
- 各予約を色分け（車両ごと or ステータスごと）
- カレンダー上の予約クリックで詳細モーダル表示
- リストビューとの切り替えボタンあり

---

### 6. 走行距離の記録

- 返却時（completed へのステータス変更時）にユーザーが入力
- 入力した距離は対象車両の累計走行距離に加算される

---

## 画面レイアウト

- **レスポンシブ対応**: スマホ・タブレット・PCすべてで使用可能
- **サイドバーナビゲーション**: PC時はサイドバー表示、スマホ時はハンバーガーメニュー
- **通知機能**: なし（必要に応じてダッシュボードで確認する方式）

## API エンドポイント一覧
